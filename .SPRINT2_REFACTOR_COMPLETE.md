## Architecture Summary (Post-Refactor)

```
Request Flow:
  Request → Validation Middleware → Chat Controller
           ↓
      Session Resolver (Create or reuse)
           ↓
      Fetch Conversation History
           ↓
      Persist User Message
           ↓
      Call LLM with Context History
           ↓
      Persist AI Response
           ↓
      Return JSON response with reply + sessionId
```

**Key Guarantees**:
- Stateless session management (lazy creation)
- History-aware LLM responses
- No feedback loops (history fetched before persisting)
- Proper error propagation to middleware
- Full observability via logging
